Что такое Git?

Введение

Википедия говорит, что "Git — распределённая система управления версиями". Если еще короче, то это ветка с версиями репозиториев. Представим дерево: cтвол - проект, ветки - версии, а листья - репозиторий.

Изначально компьютер не знает что такое Git, по этому его нужно скачать. При установке путь можно указать свой и главное не менять галочки. Чтобы проверить, установился ли Git, откройте командную строку (Пуск → Поиск → cmd) и введите git -- version.

Примеры показываются на Visual Code со встроенным терминалом, но Git позволяет работать с обычным терминалом ОС (Linux, Mac, Windows). После того как установка прошла успешно, нужно создать проект в IDE (среда программирования) и создать файл. В нашем случае index.html.

С помощью командной строки, мы должны показать git, что с этим проектом мы должны работать. В терминале прописываем команду "git init". Она инициализирует наш проект.

Поздравляю! На этом этапе можно приступать к практике.

Практика

Терминалом указываем путь в корень папки, с которой должны работать. Чтобы не заплутаться, вот команды перемещение:

  • сd test- войти в каталог под названием test
  • dir (ls для Маc и Linux) - просмотр содержимого каталога test
  • cd .. - вернуться назад

    • Пишем в терминал команду git status. Команда показывает статус файлов. В дальнейшем, мы с ним не один раз будем с ней работать.

    На данном этапе в терминале показывается, что мы должны добавить файлы, с которыми будем работать. Существует команда: git add и прописываем файт. Например, git add index.html.     Какой смысл ветки (версии)? Изначально мы работаем с веткой master, которая идет по умолчанию. Если создадим новую ветку, допустим test, и изменим код, то это будет другая версия проекта. Система удобная, если нужно оставить рабочий сайт, и создать его клон, чтобы с ним по экспериментировать. Бывает, что эксперимент не удался и можно вернуться на прежнюю версию. В ином случае, если клон работает, то можно объединить версии. Давайте по порядку.

    Команды, которые применяются к ветке:

  • git branch - просмотр, с какой веткой идет работа
  • git branch test - создать ветку с названием test
  • git branch -D test - удалить ветку с названием test

    Хорошо, а что, если получился тот результат, который нужен? Нужно объединить версии. Делает с помощью команды git merge. Прописываем команду git merge readme и у нас в ветке master есть все, чтобы было в ветке readme. На этом git не заканчивается, но тут расписана базовые команды, которые должен знать каждый.

    Все что мы делали, было локально. Но есть программисты, которые работают не с одного компьютера, а с разных. Нужно залить в сеть или скинуть. Есть варианты: "отправить по электронной почте", "отправить в меседжер", "переписать на листочек и отправить по почте". Но таким никто не пользуется, тем более последним вариантом. Для этого придумали Github.

    Что такое Github?

    Википедия говорит, что GitHub — это веб-сервис для хостинга IT-проектов и их совместной разработки.

    Значит, что каждый кто пишет код может залить его на Github и каждый желающий может его посмотреть или совместно работать с проеком.

    Регистрируемся в Github, указываем рабочую электронную почту, на которую должно придти письмо с подтверждением. Подтверждаем и заходим в профиль.

    Перед глазами у нас 4 вкладки:

  • Overview: показывает фотографию профиля, популярные репозитории и календарь активности пользователя.
  • Repositories: репозитории пользователя
  • Project: проекты пользователя
  • Packages: пакеты пользователя

    • Переходим на главную и создаем репозиторий с помощью кнопки New

    Нас встречается страничка, где мы можем настроить наш новый репозиторий:

    1. Repository Template - репозиторий, с которым мы можем работать.

    2. Owner и Repository Name - имя пользователя и его репозиторий. Если это сайт и будем публиковать на хостинг, то это будет служить доменом.

    3. Description - описание репозитория. Оно опциональное, по этому можно не писать

    4. Radiobox Public или Privat - возможность просматривать репозиторий для всех или только для определенных лиц.

    5. Add .gitignore, add a README file - опционально. README file - инструкция к репозиторию. ".gitignore " - файл, в котором будет записаны файлы или папки, которые будут игнорироваться.

    Нажимаем на "Create repository" и создаем первый репозиторий. У нас есть три способа создать содержимое репозитория:

    1. Create a new file - создать файл в репозитории

    2. Uploading an existing file - загрузить существующие файлы в репозиторий

    3. ...or create a new repository on the command line - список команд, чтобы создать и опубликовать репозиторий на хостинг.

    4. ...or push and existing repository from the command line - список команд, чтобы опубликовать репозиторий на хостинг

    5. ...or import code from another repository - импортировать код в репозиторий

    Мы познакомились c git командами и ними и опубликуем наши файлы.

    git push -u origin master - команда, которая публикует ветку в наш репозиторий.

    Как получить код, если в нем были внесены правки? Команда git pull скачивает репозиторий и переносит вIDE.

    Итог

    Мы научились пользоваться базовыми командами git, создавать объекты, создавать версии, переключаться между ними, создавать репозитории, публиковать проект и принимать проект. Какие команды для этого нужны:

  • git init - инициализировать проект
  • git status - посмотреть статус проекта
  • git add index.html - добавить файл index.html в git
  • git add . - добавить все файлы в git
  • git rm --cached index.html - изьять файл index.html с git
  • файл .gitignore - файл, в котором хранится название папок и файлов, которые игноряться
  • git branch - просмотр веток
  • git branch test - создать ветку test
  • git branch -D test - удалить ветку test
  • git checkout test - переместиться на ветку test
  • git checkout -b new test - создать ветку test и работать с ней
  • git merge readme - объеденение ветки readme с master
  • git push -u origin master - опубликовать ветку master на github
  • git push - опубликовать код на хостинг
  • git pull - взять код с репозитория
